Spring Boot 如何实现整合谷歌令牌

本文说明如何在 Spring Boot + Vue 后台管理系统中接入 Google Authenticator（TOTP 动态令牌），实现「创建用户绑定 + 登录强制校验」。

1. 功能目标

场景 说明
新增用户 自动生成密钥 mfa，管理端展示二维码供扫码绑定
用户登录 除账号密码外，必须输入 6 位动态令牌
历史用户 上线前需批量补齐 mfa，否则无法登录

4.4 登录：强制校验动态令牌

登录 DTO 沿用原有 code 字段，语义改为「谷歌令牌 6 位数字」。

要点：

  • 去掉图形验证码校验（若原登录用 Redis 存验证码，可删除相关逻辑）。
  • 密码正确 ≠ 登录成功，必须 TOTP 也通过。
  • 异常信息区分：未绑定 / 格式错误 / 令牌错误。

5. 前端实现（Vue2）

5.1 登录页

  • 输入框文案改为「谷歌令牌」。
  • 仍提交字段 code（6 位数字）。
  • 不再请求图形验证码接口。

5.2 新增用户：本地生成 otpauth + 二维码

不要使用 https://www.google.com/chart?... 生成二维码（该服务已不可用，会 404）。

推荐做法（与参考项目一致）：

  1. 安装本地二维码组件：npm install vue-qrcode-component
  2. 准备 Base32 编码工具（前端 utils/base32.js）
  3. 用户输入用户名时，前端生成密钥并渲染二维码

7. 接口约定小结

接口 方法 说明
/auth/login POST body 含 username、password、code（动态令牌）
/api/users POST body 含用户信息及 mfa（可选，后端可兜底生成）

8. 常见问题

Q：二维码显示不出来，Network 里 google.com/chart 404？
A：改用前端 vue-qrcode-component 渲染 otpauth:// 字符串，不要依赖 Google Chart API。

Q：令牌总是错误？
A：检查服务器 NTP 时间；确认入库 mfa 与扫码绑定密钥一致；适当增大 window_size。

Q：老用户登录提示未绑定？
A：该用户 mfa 为空，需先跑迁移 SQL 或在管理端重新生成密钥。

Q：MySQL 执行 ADD COLUMN 报语法错误？
A：低版本不支持 IF NOT EXISTS，改用先查 information_schema.COLUMNS 再决定是否 ALTER。

9. 安全建议

  • mfa 为敏感字段，日志中禁止打印完整密钥与动态码
  • 重置密钥后旧令牌立即失效
  • 可配合登录失败次数限制，降低暴力猜码风险
  • 生产环境务必使用 HTTPS

完成以上步骤后，Spring Boot 与 Google Authenticator 的整合即可投入使用。
