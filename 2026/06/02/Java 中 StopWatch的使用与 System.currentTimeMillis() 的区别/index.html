Java 中 StopWatch的使用与 System.currentTimeMillis() 的区别

StopWatch 作用

在性能调优和问题排查时，统计方法执行耗时是非常常见的需求。
最常见的方式是用 System.currentTimeMillis()，但在 Spring 框架里还提供了更方便的工具类 —— StopWatch。

本文将对比两者的区别，并给出示例。

使用 System.currentTimeMillis()

输出：

特点

  • ✅ 优点：无依赖，写法简单，适合快速验证。
  • ❌ 缺点：
    • 只能记录单段耗时；
    • 需要手动计算差值；
    • 不支持分步骤统计，可读性差。

使用 StopWatch（Spring 工具类）

Spring 提供了 org.springframework.util.StopWatch，可以对多个任务进行分段计时，并生成详细报告。

对比总结

对比项 System.currentTimeMillis() StopWatch
来源 JDK 原生 Spring 工具类
精度 毫秒 纳秒（更精确）
功能 单段耗时统计 多任务分段统计
输出 纯数字 报表，直观可读
依赖 无 Spring 框架
适用场景 临时简单耗时计算 性能调优、分步骤统计

总结

  • 如果只是快速测量某个方法的执行时间：System.currentTimeMillis() 足够。
  • 如果需要分析业务流程中多个阶段的耗时分布：StopWatch 更直观、功能更强大。
  • 在日志中输出 StopWatch.prettyPrint() 的表格，可以快速定位性能瓶颈。
